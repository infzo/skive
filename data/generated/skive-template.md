# Skive

帮助用户快速创建符合最佳实践的高质量 Skill。

## 目标

基于已分析的设计范式，引导用户创建结构规范、内容完整、易于维护的 Skill 文件。

## 使用方式

### 交互模式

执行 `/skive`，系统会引导你完成：
1. 需求分析
2. 模板选择
3. 内容填充
4. 质量验证
5. 文件生成

### 快速模式

执行 `/skive --template=<模板名> --name=<Skill名称>`

直接使用指定模板创建 Skill

## 可用模板

### 1. simple（简单任务模板）

适用于：单一明确任务、步骤少于 5 步

**使用**：`/skive --template=simple --name=<名称>`

### 2. workflow（工作流模板）

适用于：多阶段任务、需要条件判断、步骤 5-10 步

**使用**：`/skive --template=workflow --name=<名称>`

### 3. complex（复杂系统模板）

适用于：企业级应用、需要详细文档、长期维护

**使用**：`/skive --template=complex --name=<名称>`

## 创建流程

### 步骤 1：需求分析

回答以下问题：
1. Skill 的核心目标是什么？
2. 用户在什么场景下使用？
3. 需要什么前置条件？
4. 输出什么结果？
5. 有哪些异常情况需要处理？

### 步骤 2：模板选择

根据任务复杂度选择模板：

| 任务类型 | 推荐模板 | 特点 |
|---------|---------|------|
| 工具封装 | simple | 快速定义，简洁明了 |
| 数据处理流程 | workflow | 多阶段，有分支 |
| 系统集成 | complex | 完整文档，长期维护 |

### 步骤 3：内容填充

按模板结构填充内容，确保：
- 目标明确可测量
- 步骤具体可执行
- 输入输出定义清晰
- 包含错误处理

### 步骤 4：质量验证

使用质量检查清单验证：
- [ ] 元数据完整
- [ ] 目标明确
- [ ] 步骤可执行
- [ ] 输出定义清晰

### 步骤 5：文件生成

将 Skill 保存到 `skills/` 目录

## 设计原则

基于已分析的 10 个优秀 Skill，遵循以下设计原则：

1. **结构标准**：使用 YAML frontmatter + 标准章节
2. **目标明确**：定义清晰的使用时机和目标
3. **步骤适中**：粗粒度概述 + 关键细节展开
4. **示例丰富**：提供可直接复制使用的示例
5. **错误处理**：说明常见错误和处理方式

## 输出位置

生成的 Skill 文件存放在 `skills/generated/` 目录

## 质量保证

每个生成的 Skill 都会自动通过质量检查清单验证。
