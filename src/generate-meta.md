## 元技能定义

**元技能**是一种特殊的 Skill，其核心功能是帮助用户创建其他高质量 Skill。它封装了 Skill 设计的最佳实践、通用模式和验证标准，使 Skill 创建过程标准化、高效且可靠。

### 元技能的三个核心能力
1. **流程指导**：提供从需求到 Skill 文件的完整创建流程
2. **模板选择**：根据场景推荐合适的 Skill 结构模板
3. **质量保证**：通过检查清单验证 Skill 的完整性和可用性

## 输入

- **前置条件**：已执行 `analyze-patterns`，存在范式分析结果
- **数据来源**：
  - `data/analyzed/pattern-summary.md`（范式总结）
  - `data/analyzed/common-patterns.md`（常见范式详情）
  - `data/analyzed/best-practices.md`（最佳实践清单）

## 执行步骤

### 步骤 1：提取关键范式

**目标**：从分析结果中提炼可复用的设计模式

**操作**：
1. 读取 `data/analyzed/pattern-summary.md`
2. 提取以下三类范式：
   - **结构范式**：Skill 文件的标准组织方式
   - **触发范式**：常见的触发条件和命令设计
   - **执行范式**：步骤编排和错误处理模式

**输出**：范式清单列表，例如：
```markdown
## 结构范式
- 扁平式结构：适用于简单任务
- 分层式结构：适用于复杂工作流
- 混合式结构：结合前两者优势

## 触发范式
- 命令触发：`/skill-name`
- 关键词触发：匹配特定上下文
- 条件触发：基于环境或状态

## 执行范式
- 线性步骤：顺序执行
- 条件分支：根据情况选择路径
- 循环迭代：重复执行直到完成
```

### 步骤 3：创建质量检查清单

**目标**：建立 Skill 质量验证标准

**检查维度**：

#### 1. 完整性检查
- [ ] 元数据完整（name, description, trigger）
- [ ] 目标明确可测量
- [ ] 输入输出定义清晰
- [ ] 步骤覆盖完整流程

#### 2. 可执行性检查
- [ ] 每个步骤可独立执行
- [ ] 步骤间的依赖关系明确
- [ ] 无循环依赖或死锁
- [ ] 错误处理覆盖关键失败点

#### 3. 可维护性检查
- [ ] 命名语义化、无歧义
- [ ] 结构清晰、易于理解
- [ ] 有版本控制和更新记录
- [ ] 包含使用示例（推荐）

#### 4. 最佳实践检查
- [ ] 触发条件避免冲突
- [ ] 遵循单一职责原则
- [ ] 可组合、可扩展
- [ ] 有回滚或恢复机制

**输出文件**：`quality-checklist.md`

### 步骤 4：生成元技能 Skill 文件

**目标**：创建可执行的元技能 Skill

**文件结构**：
```markdown
---
name: skive
description: 创建高质量 Skill 的 Skill
trigger: /skive
---

## 目标
帮助用户快速创建符合最佳实践的高质量 Skill

## 使用方式

### 交互模式
执行 `/skive`，系统会引导你完成：
1. 需求分析
2. 模板选择
3. 内容填充
4. 质量验证
5. 文件生成

### 快速模式
执行 `/skive --template=<模板名> --name=<Skill名称>`
直接使用指定模板创建 Skill

## 可用模板
- `simple`：简单任务模板
- `workflow`：工作流模板
- `complex`：复杂系统模板

## 输出位置
生成的 Skill 文件存放在 `skills/generated/` 目录

## 质量保证
每个生成的 Skill 都会自动通过质量检查清单验证
```

**输出文件**：`skive-template.md`

### 步骤 5：验证机制

**目标**：确保元技能能够正确生成 Skill

**验证流程**：

1. **模拟测试**：使用元技能创建一个测试 Skill
   - 选择模板：简单任务模板
   - 测试场景：创建一个"代码格式化"Skill
   - 预期结果：生成完整的 Skill 文件

2. **质量验证**：运行质量检查清单
   - 检查所有必需字段
   - 验证可执行性
   - 确认符合最佳实践

3. **实际执行**：运行生成的测试 Skill
   - 验证触发是否正常
   - 检查执行流程是否符合预期

**验证通过标准**：
- 生成的 Skill 文件格式正确
- 质量检查清单全部通过
- 测试 Skill 能够正常执行

## 注意事项

1. **通用性**：元技能应适用于各种类型的 Skill 创建
2. **灵活性**：提供多种模板选项，支持自定义扩展
3. **演进性**：模板应随着最佳实践的积累而更新
4. **易用性**：交互模式应引导用户完成所有必要步骤
5. **质量优先**：生成的 Skill 必须通过质量验证
